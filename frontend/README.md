**CyberV2X Orchestrator** is a web-based platform designed to accelerate research in **5G V2X Cybersecurity**. It serves as a "Tool Paper" interface that abstracts the complexity of configuring SUMO, OMNeT++, Veins, and Simu5G.

Researchers can visually design scenarios, inject jamming attacks, and automatically generate valid configuration bundles (`.zip`) ready to run in the **[B5GCyberTestV2X Framework](https://b5gcybertestv2x.hshl.de/)**.

## üöÄ Key Features

### 1. Multi-Level Simulation Workflow

- **‚ö° Quick Sim (Simple Mode):** For rapid statistical data generation. Abstracts parameters into "Low/High Intensity" and automatically distributes Jammers based on fleet density (10%).
- **üõ†Ô∏è Advanced Sim (Global Control):** Manual placement of infrastructure (RSUs, Static Jammers) and global network parameter configuration (Tx Power, Bitrate).
- **üß† Expert Mode (Granular Engineering):** Click-to-route logic. Define individual start/end points for specific vehicles and configure unique 5G PHY layers (Numerology, Bandwidth) per node.

### 2. Scientific Validity

- **Backend Engine:** Uses `sumolib` for precise Geo-to-Cartesian coordinate conversion.
- **Realistic Traffic:** Generates background traffic using SUMO's `randomTrips.py` mixed with manual fixed routes.
- **Correct Protocols:** Implements `VoIPSender` (Uplink) and `VoIPReceiver` applications to simulate realistic UDP data flows in Simu5G.

### 3. Analytics Dashboard

- **Visual Validation:** Upload `.vec` and `.sca` files generated by OMNeT++.
- **Charts:** Automatic plotting of **SINR** (Signal-to-Interference-plus-Noise Ratio) and **Throughput** over time to visualize attack impact.

## üì¶ Installation & Usage

### Prerequisites

- Node.js (v16+)
- Python (v3.10+)
- **SUMO (Simulation of Urban MObility)** installed and available in your PATH (Required for route generation).

### 1. Backend Setup

```bash
cd backend
python -m venv venv
source venv/bin/activate  # or venv\Scripts\activate on Windows
pip install -r requirements.txt

# Start the API
uvicorn app.main:app --reload
```
